# TOO «MOOL» компаниясының қысқаша сипаттамасы

TOO «MOOL» (бұдан әрі – MOOL) – 2021 жылы Алматы қаласында құрылған шағын кәсіпорын. Компания B2B оптовая платформа ретінде жұмыс істейді және өндірушілер мен жеткізушілерді саудагерлермен онлайн түрде байланыстырады. MOOL қызметі электронды коммерция және бағдарламалық қамтамасыз ету саласына жатады. Платформа арқылы клиенттер әртүрлі жеткізушілердің баға ұсыныстары мен жеткізу шарттарын салыстырып, өздеріне тиімді өнімді таңдай алады.

Компанияның бизнес-моделі қарапайым: платформада жасалған әрбір мәміледен 3–5% көлемінде комиссия алады. MOOL сайтына ай сайын шамамен 5 000 жаңа пайдаланушы кіреді, олардың бір бөлігі нақты тапсырыс рәсімдейді. Қазіргі таңда компания Орталық Азия нарығына шығуды және серіктестер санын арттыруды мақсат етіп отыр.

Компанияда интернет-сату және IT-жобалар саласында тәжірибесі бар мамандарға басымдық беріледі. Қызметкерлерден қазақ және орыс тілдерін білу, цифрлық сауда құралдарымен жұмыс істей алу және командада тиімді қызмет атқару талап етіледі. MOOL өз клиенттеріне ыңғайлы онлайн-сауда алаңын ұсынып, сенімді ақпарат арқылы тиімді шешім қабылдауға көмектесетін заманауи кәсіпорын болып табылады.

## Сұхбат өтті

Практика басталар алдында студентпен алдын ала техникалық әңгімелесу жүргізілді. Әңгіме барысында жоба менеджері SmartCourse жүйесінің архитектурасын — Go (Gin) негізіндегі backend пен Next.js негізіндегі frontend арасындағы өзара байланысты ұйымдастыру, сондай-ақ AI-қуатты білім беру платформасын дамытуға қатысты негізгі тапсырмаларды түсіндірді. Негізгі басымдық — Next.js + React технологиялары арқылы заманауи веб-қосымшаның интерфейсін қалыптастыру, Go + Gin фреймворкімен RESTful API әзірлеу, PostgreSQL деректер қорымен жұмыс істеу және OpenRouter/Gemini AI сервистерін интеграциялау арқылы оқытушылар мен студенттерге қолайлы тәжірибені қамтамасыз ету болды.

## 3. Компания мәселелерінің қысқаша сипаттамасы.

MOOL компаниясы SmartCourse платформасын дамыту барысында бірнеше негізгі мәселелерге тап болды: (1) білім беру нарығында AI технологияларын қолданатын бәсекелес платформалар пайда болуда, сондықтан SmartCourse-тың бәсекеге қабілетті болып қалуы үшін AI функционалын үнемі жаңартып, жаңа мүмкіндіктер қосу қажет. (2) Платформаның масштабталуы мәселесі — студенттер мен оқытушылар саны артқан сайын backend жүктемесін тиімді басқару, Redis кэштеу мен асинхронды кезектерді дұрыс баптау қиындық туғызады. Маңызды мәселелердің бірі — OpenRouter және Gemini AI сервистерінің rate-limit шектеулері мен құны, яғни тегін деңгейдегі API лимиттері жоғары жүктемеде жеткіліксіз болуы мүмкін. Антиплагиат жүйесінің TF-IDF алгоритмі қысқа мәтіндерде дәлдігі төмен болуы мүмкін, бұл жалған позитивтерге әкеледі. MOOL тап болған тағы бір мәселе — көптілділік қолдауы, яғни қазақ, орыс және ағылшын тілдерінде AI жауаптарының сапасын бірдей деңгейде ұстау қиын. Сондай-ақ, компания деректер қауіпсіздігін қамтамасыз ету мәселесіне де көңіл бөледі — студенттердің жеке деректері мен тапсырмаларын қорғау үшін JWT аутентификация мен MinIO presigned URL-дерін үнемі жетілдіру қажет.

## 4. Таңдалған диплом жұмысы тақырыбының себептері мен өзектілігін түсіндіру, көрсетілген проблемаларды шешу, компания үшін пайдалылығы аясында.

MOOL компаниясы тап болған мәселелердің ішінен мен AI негізінде білім беру платформасын толық циклді әзірлеуді таңдадым. Компания менеджері менен SmartCourse жүйесінің жұмыс прототипін 03.04.2026 күніне дейін күтеді, егер нәтиже қанағаттандырарлық болса, платформаны нақты оқу орындарында енгізу жоспарланып отыр. Бұл тақырып өзекті, себебі Қазақстандағы білім беру саласында AI көмегімен тапсырмаларды автоматты бағалау, антиплагиат тексеру және жеке оқыту ассистенті сияқты мүмкіндіктер әлі толық қамтылмаған. Университет бекіткен менің диплом жұмысымның тақырыбы өзгертілген жоқ, ол бастапқыда SmartCourse платформасын әзірлеуге бағытталған болатын. Жоба Go + Next.js + AI технологияларын біріктіретін күрделі жүйе болғандықтан, компания үшін нақты пайда — оқытушылардың жүктемесін азайту, студенттердің білім сапасын арттыру және оқу процесін автоматтандыру.

## 5. Алынған практикалық тәжірибе және компаниямен одан әрі ынтымақтастық жасау бойынша ұсынымдар.

MOOL компаниясы — технологиялық жағынан дамыған және болашағы зор ұйым. Компанияда Go, Next.js, AI сервистері сияқты заманауи технологиялармен жұмыс істеу мүмкіндігі бар және олар білім беру саласында платформаны одан әрі кеңейтуді жоспарлауда. Менеджерлер студенттермен тиімді жұмыс істеді, жаңа идеяларды талқылауға ашық болды, мұнда мен backend архитектура, AI интеграция, антиплагиат жүйесі және фронтенд әзірлеу бойынша көп практикалық тәжірибе алдым. Компания жетекшілеріне менің SmartCourse жобасы бойынша атқарған жұмысым ұнады, олар болашақта басқа да студенттерді практикаға қабылдауға мүдделі. Осылайша, менің ойымша, MOOL компаниясымен ынтымақтастықты нығайту және жалғастыру қажет.

## Бірінші апталық әрекеттер (09.02.2026 – 15.02.2026)

**Мақсат:**
Go тілінде Gin фреймворкін пайдаланып REST API жобасын инициализациялау және Docker Compose арқылы инфрақұрылымды толық баптау.

**Негізгі міндеттер:**
- Go + Gin фреймворкімен REST API жобасын құру
- Жобаның модульдік құрылымын анықтау (internal/models, internal/repository, internal/services, internal/delivery, internal/routes)
- Docker Compose арқылы PostgreSQL, Redis, MinIO сервистерін баптау
- GORM ORM кітапханасын қосу және деректер қоры миграцияларын жазу

**Қолданылған технологиялар:**
- Go 1.22, Gin фреймворкі — HTTP маршруттау
- GORM ORM — PostgreSQL-мен жұмыс
- Docker Compose — контейнерлік инфрақұрылым
- golang-migrate — деректер қоры миграциялары

**REST API жобасын инициализациялау (09.02.2026 – 11.02.2026)**

Орындалған жұмыстар:
- `cmd/main.go` — қосымшаның кіру нүктесі, Gin engine іске қосу
- `internal/routes/routes.go` — `SetupRoutes()` функциясы: барлық маршруттарды, middleware-ді, репозиторийлер мен сервистерді инициализациялау
- `internal/models/` — деректер қоры модельдері: `User`, `Course`, `Assignment`, `Grade` құрылымдарын анықтау
- Жобаның Clean Architecture модульдік құрылымын қалыптастыру: Repository → Service → Handler (Delivery)

**Docker Compose баптау және миграциялар (12.02.2026 – 15.02.2026)**

Орындалған жұмыстар:
- `docker-compose.yml` файлын жасау: PostgreSQL 17, Redis 7, MinIO, Prometheus, Grafana сервистерін анықтау
- `internal/db/db.go` — `InitDB()` функциясы: PostgreSQL-ге GORM арқылы қосылу, ортаның айнымалыларын оқу, миграцияларды автоматты қолдану
- `internal/db/migrations/` — SQL миграция файлдары: `users`, `courses`, `course_students`, `assignments`, `grades` кестелерін құру
- Volume-дерді баптау: `postgres_data`, `redis_data`, `minio_data` — деректерді сақтау

**Бірінші апта нәтижелері:**
REST API жобасы сәтті инициализацияланды. Docker Compose арқылы барлық сервистер бір `docker-compose up` командасымен іске қосылатын болды. GORM миграциялары деректер қорын автоматты құрады.

## Екінші апталық әрекеттер (16.02.2026 – 22.02.2026)

**Мақсат:**
JWT токен негізінде қауіпсіз аутентификация жүйесін құру және рөлдік авторизацияны іске асыру.

**Негізгі міндеттер:**
- Тіркелу (Register) және кіру (Login) эндпойнттарын жасау
- Парольді bcrypt алгоритмімен хэштеу
- JWT токен генерациясы мен тексеру
- AuthMiddleware және RoleMiddleware жасау
- CORS баптау

**Қолданылған технологиялар:**
- golang-jwt/jwt/v5 — JWT токен генерациясы
- golang.org/x/crypto/bcrypt — парольді хэштеу
- Gin middleware — сұрау өңдеу тізбегі

**JWT аутентификация жүйесін жасау (16.02.2026 – 18.02.2026)**

Орындалған жұмыстар:
- `internal/auth/handler.go` — `AuthHandler` құрылымы: `Register()` және `Login()` функцияларын жазу
- `RegisterRequest` — username, password, role валидациясы (`binding:"required,min=6"`)
- `LoginRequest` — bcrypt.CompareHashAndPassword арқылы парольді тексеру
- JWT токен: UserID, Username, Role қосу, 24 сағат мерзім, HS256 қолтаңба алгоритмі
- `internal/models/user.go` — `User` моделі: ID, Username, Password, Role (admin/teacher/student) өрістерін анықтау

**Middleware және CORS баптау (19.02.2026 – 22.02.2026)**

Орындалған жұмыстар:
- `internal/auth/middleware.go` — `AuthMiddleware()`: Authorization header-ден Bearer токенді оқу, `jwt.ParseWithClaims` арқылы тексеру, `user_id`, `username`, `role` мәндерін Gin контекстіне жазу
- `RoleMiddleware(role)` — контекстегі рөлді тексеру, қол жеткізуді шектеу
- `AdminMiddleware()` — тек admin рөлі үшін қол жеткізу
- CORS middleware: `Access-Control-Allow-Origin`, `Allow-Methods`, `Allow-Headers` баптау — frontend (localhost:3000) мен backend арасындағы байланыс
- `internal/middleware/metric.go` — MetricMiddleware: сұрау уақытын, статус кодтарын логтау

**Екінші апта нәтижелері:**
Аутентификация жүйесі толық жұмыс істейтін болды. Үш рөл (Admin, Teacher, Student) бөлек маршруттармен қорғалды. JWT токен 24 сағат бойы жарамды болады.

## Үшінші апталық әрекеттер (23.02.2026 – 01.03.2026)

**Мақсат:**
Курстар мен тапсырмалар модулін Clean Architecture принципі бойынша толық әзірлеу.

**Негізгі міндеттер:**
- Course моделі: CRUD операциялары, студенттерді курсқа қосу/шығару
- Assignment моделі: эссе/тест түрлері, критерийлер, тест сұрақтары
- Repository → Service → Handler қабаттарын жасау
- Рөлдерге арналған REST API маршруттарын құру

**Қолданылған технологиялар:**
- GORM ORM — реляциялық байланыстар (belongsTo, many2many)
- Gin — RESTful маршруттау
- JSON — критерийлер мен сұрақтарды сақтау

**Course модулін әзірлеу (23.02.2026 – 25.02.2026)**

Орындалған жұмыстар:
- `internal/models/course.go` — `Course` құрылымы: Title, Description, TeacherID өрістері, Teacher (belongsTo), Students (many2many), Assignments (hasMany) байланыстары

MinIO файл жүктеу/жүктеп алу presigned URL арқылы қауіпсіз жұмыс істеді. Барлық сервистер Docker Compose арқылы бір ортада сәтті deploy етілді. Тестілеу, қателерді түзету және код оңтайландыру аяқталды. SmartCourse платформасы толық жұмыс істейтін прототип ретінде дайын болды.
